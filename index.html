Static Adaptive Noise Filter System on Zynq-7020

Challenge

LMS, NLMS, and RLS adaptive filters are compute-intensive and each suits different noise conditions. A unified hardware system was needed that could host all three and switch between them in real time on the Zynq-7020 FPGA.

Solution

Designed LMS, NLMS, and RLS accelerators in Vitis HLS and exported them as AXI-compatible IP cores to Vivado. Built a top-level block design instantiating all three IPs simultaneously, with a GPIO peripheral selecting the active filter at runtime based on noise conditions. Integrated AXI DMA and DDR controllers; validated on hardware with Vivado ILA and JTAG.

Outcome
<30%LUTs used — all three filter datapaths fit comfortably
~25%DSP slices — headroom remaining for additional processing

Each algorithm offers distinct trade-offs: LMS excels in simplicity and power; NLMS in stability under varying conditions; RLS in fastest convergence for performance-critical paths.

Live Filter Switching via Dynamic Function eXchange (DFX)

Challenge

The static design instantiates all three filter datapaths simultaneously, consuming ~25% DSP slices and causing routing congestion that limits timing slack. For embedded and battery-powered applications, a leaner architecture that loads only one filter at a time was needed — without halting the FPGA.

Solution

Implemented Xilinx Dynamic Function eXchange (DFX) by creating two pblocks as reconfigurable regions on the Zynq-7020. Generated partial bitstreams for each filter variant (LMS, NLMS, RLS) and designed a parent configuration holding the static logic. Filters are swapped at runtime — only one active filter resides in the fabric at any time — with CDC issues resolved across the pblock boundaries.

Outcome
5%DSP48 used in DFX parent — vs ~25% in static design
0System downtime during filter switch — FPGA never halted

Reduced logic density improved timing margin at 100 MHz target. Lower dynamic power makes this architecture ideal for hearing-aid devices and battery-powered audio systems.

AXI4 Interconnect & DMA Verification Environment

Challenge

An AXI4 DMA IP with complex burst modes and ordering rules needed exhaustive pre-silicon verification — corner cases that directed tests alone would never reach.

Solution

Built a reusable AXI VIP with UVM drivers, protocol-aware monitors, and a scoreboard. Applied constrained-random stimulus targeting burst boundaries, out-of-order responses, and all handshake edge cases. SVA assertions continuously checked AWVALID/READY handshake, burst ordering, and BVALID response correctness.
